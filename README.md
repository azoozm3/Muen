# Mu'en

Mu'en is a full-stack medical support platform built with React, Vite, Express, MongoDB, and Mongoose.

## Main roles

- Patient
- Doctor
- Nurse
- Volunteer
- Admin

## Core flows

- Emergency requests
- Doctor appointments and online consultations
- Nurse requests
- Volunteer requests
- Ratings and reviews
- Admin dashboard and user management
- PayPal payments for paid services
- Zoom online appointments

## Tech stack

- Frontend: React, Vite, Wouter, TanStack Query, Tailwind CSS
- Backend: Node.js, Express, MongoDB, Mongoose
- Auth/session: express-session + connect-mongo
- Payments: PayPal
- Video meetings: Zoom

## Run locally

1. Copy `.env.example` to `.env`.
2. Add your real MongoDB connection string and any optional PayPal/Zoom credentials.
3. Install dependencies.
4. Start the app.

```bash
npm install
npm run dev
```

Default local server:

- App and API: `http://localhost:5000`
- Health check: `http://localhost:5000/api/health`

## Build for production

```bash
npm run build
npm start
```

## Important notes

- Do not commit or share `.env`.
- Do not include `node_modules`; run `npm install` after extracting the project.
- `dist` is generated by `npm run build`, so it is not included in the clean source package.
- The project no longer auto-seeds fake doctors or reviews at runtime.
- PayPal can run in mock mode locally when credentials are missing.
